Tailwind Breakout Grid

A flexible, fluid grid layout system for content that breaks out of its container. Press Cmd+G to toggle the visualizer.

View masthead markup 
<header class="grid-cols-breakout bg-slate-900">
  <div class="col-full-limit px-gap py-4 flex justify-between">
    <div>Logo</div>
    <nav>Links...</nav>
  </div>
</header>

<!-- col-full-limit: full width with max-width cap
     px-gap: responsive padding that matches grid gaps -->

Basic Column Widths

Each column width serves a different purpose. Narrow is for readable text, content for general layout, and wider columns for emphasis.

.col-full Edge to edge
.col-full-limit Full width with max-width constraint
.col-feature Extra wide for heroes & images
.col-popout Slightly wider than content
.col-content Standard content width (default)
.col-narrow Optimal reading width (40-50rem)
View markup 
<!-- Container with breakout grid -->
<div class="grid-cols-breakout">

  <!-- Column utilities (widest to narrowest) -->
  <div class="col-full">Edge to edge</div>
  <div class="col-full-limit">Full with max-width</div>
  <div class="col-feature">Extra wide</div>
  <div class="col-popout">Slightly wider</div>
  <div class="col-content">Standard width</div>
  <div class="col-narrow">Reading width</div>

  <!-- Default: items without col-* use defaultCol setting -->
  <p>Uses default column (content by default)</p>

</div>

Padding Utilities

Responsive padding that scales with the grid. Use on full-width sections to maintain consistent spacing.

.col-full .p-gap

Padding matches the responsive grid gap on all sides.

.col-full .p-breakout

Larger padding that aligns inner content with .col-content edges.

.col-feature .p-gap

Works on any column width, not just full.

<!-- Padding utilities:
     px-breakout  - Aligns inner content to content column
     px-gap       - Uses the responsive --gap value
     p-breakout   - All sides
     py-breakout  - Top and bottom only
-->

Left & Right Alignment

Span from one edge to a specific column boundary for asymmetric layouts.

.col-feature-left Left edge to feature boundary
Feature boundary to right edge .col-feature-right
.col-center-left Left edge to center point
Center point to right edge .col-center-right
View markup 
<!-- Left-aligned spans (left edge to boundary) -->
<div class="col-feature-left">Left edge to feature boundary</div>
<div class="col-popout-left">Left edge to popout boundary</div>
<div class="col-content-left">Left edge to content boundary</div>
<div class="col-center-left">Left edge to center point</div>

<!-- Right-aligned spans (boundary to right edge) -->
<div class="col-feature-right">Feature boundary to right edge</div>
<div class="col-popout-right">Popout boundary to right edge</div>
<div class="col-content-right">Content boundary to right edge</div>
<div class="col-center-right">Center point to right edge</div>

<!-- For custom spans, use start/end utilities -->
<div class="col-start-content col-end-feature">
  Content start to feature end
</div>

CSS Subgrid

Use .grid-cols-breakout-subgrid on nested elements to inherit the parent grid's tracks. Children can align to any named line.

.col-feature - spans the feature area
.col-narrow - aligns to narrow!
.col-content - aligns to content!
View markup 
<div class="col-feature-right grid-cols-breakout-subgrid">
  <div class="col-feature">
    Spans the feature area
  </div>
  <div class="col-narrow">
    Aligns to narrow!
  </div>
  <div class="col-content">
    Aligns to content!
  </div>
</div>
.col-popout - popout width
.col-content - content alignment
View markup 
<div class="col-feature-left grid-cols-breakout-subgrid">
  <div class="col-popout">
    Popout width
  </div>
  <div class="col-content">
    Content alignment
  </div>
</div>

Browser Support: CSS Subgrid has ~90% browser support as of January 2025. Check current support

Nested Grids

Use breakout-to-* modifiers when nesting grids inside constrained containers.

Parent: .col-feature

.col-feature fills the container
.col-content has proper margins
.col-narrow for readable text

Nested: .grid-cols-breakout .breakout-to-feature

View markup 
<!-- Parent container constrains width -->
<div class="col-feature">

  <!-- Nested grid collapses outer tracks -->
  <div class="grid-cols-breakout breakout-to-feature">
    <div class="col-feature">Fills the container</div>
    <div class="col-content">Has proper margins</div>
    <div class="col-narrow">For readable text</div>
  </div>

</div>

<!-- Available modifiers:
     breakout-to-narrow   - Collapse all tracks
     breakout-to-content  - Keep content margins
     breakout-to-popout   - Keep popout + content
     breakout-to-feature  - Keep feature + popout + content
-->

<article class="grid-cols-breakout">

  <!-- Header at content width -->
  <header class="col-content">
    <h1>Article Title</h1>
  </header>

  <!-- Body text at narrow (reading width) -->
  <p class="col-narrow">
    Readable paragraph text...
  </p>

  <!-- Wide image at feature width -->
  <figure class="col-feature">
    <img src="hero.jpg" alt="Hero" />
  </figure>

  <!-- Blockquote pops out slightly -->
  <blockquote class="col-popout bg-blue-50">
    "Pull quote for emphasis..."
  </blockquote>

  <!-- Full-width CTA band -->
  <div class="col-full bg-slate-900">
    <div class="px-breakout py-12">
      Inner content aligns with col-content
    </div>
  </div>

</article>
